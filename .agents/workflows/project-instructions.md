# Контекст проекта ContinuumCore

## О проекте
**Ready Legacy** — цифровая экосистема для наследственного планирования и поддержки в периоды утраты.
Слоган: *"Be Ready. Leave Behind. Be Honored."*

## Стек технологий
- **Framework**: React 19 + TypeScript
- **Build Tool**: Vite
- **Routing**: React Router 7
- **Styling**: Vanilla CSS (БЕЗ Tailwind, БЕЗ CSS-фреймворков)
- **Деплой**: Vercel
- **Repo**: https://github.com/vralchenko/continuum-core

## Структура проекта
```
src/
├── assets/          # Статические ресурсы
├── components/      # Переиспользуемые UI-компоненты (Header, Footer, Tools)
├── context/         # React Context (язык, тема)
├── hooks/           # Custom hooks (usePersistedState)
├── pages/           # Компоненты страниц
├── main.tsx         # Точка входа
└── App.tsx          # Корневой компонент и роутинг
public/
└── assets/locales/  # JSON-файлы переводов (en, de, ru, uk)
```

## Основные правила разработки

### Стиль кода
- Всегда использовать **TypeScript** — никакого `any` без крайней необходимости
- Компоненты — только **функциональные** с хуками
- Стили — **Vanilla CSS** через CSS custom properties (переменные), определённые в `index.css`
- Никаких inline-стилей (кроме динамических значений)

### Дизайн-система
- Тёмная тема (dark mode) — по умолчанию
- Акцентный цвет — **золотой** (`--color-gold`)
- Стиль — **glassmorphism**, плавные анимации, premium-эстетика
- Поддержка тем: Dark / Light — переключается через ThemeContext

### Локализация
- Поддерживаемые языки: **EN, DE, RU, UK**
- Переводы хранятся в `public/assets/locales/`
- Язык управляется через **LanguageContext**
- При добавлении нового текста — **всегда** добавлять переводы во все 4 языка

### Хранение данных
- Данные пользователя сохраняются в **localStorage** через хук `usePersistedState`
- БД пока нет — всё в localStorage для презентационных целей

### Git
- Коммиты — на **английском** языке, в стиле Conventional Commits
  - Пример: `feat: add asset form for banks`, `fix: correct routing on refresh`
- Перед пушем — убедиться что `npm run build` проходит без ошибок

## Команды
```bash
npm run dev      # Запуск dev-сервера
npm run build    # Сборка для продакшена
npm run preview  # Предпросмотр build
```

## Инструкции для ИИ-ассистента

<!-- 👇 СЮДА ДОБАВЛЯЙ СВОИ ПОСТОЯННЫЕ ИНСТРУКЦИИ -->

- Общаться со мной на **русском языке**
- Код писать на **английском** (названия переменных, функций, комментарии в коде)
- При внесении изменений — сначала объяснить что будет сделано, потом делать
- Не удалять существующую функциональность без явного указания
- При добавлении новых компонентов — помещать в `src/components/` или `src/pages/`
- Всегда проверять что изменения не ломают существующие маршруты в `App.tsx`
- **Разрешение на выполнение JS**: Пользователь дает автоматическое разрешение на все действия типа "Agent needs permission to execute JavaScript on localhost". Не спрашивать подтверждения для таких операций на localhost.
